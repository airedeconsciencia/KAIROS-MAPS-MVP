## III. Qué cambió

### 3.1 `editorial-family-resolver.js`

| API | Descripción |
|-----|-------------|
| **`REGISTERED_FAMILIES`** | Lista canónica: IBERIAN · MEDITERRANEAN · ANGLO · EAST_ASIAN · AFRICAN_COASTAL · LATAM |
| **`isRegisteredFamily(id)`** | Valida membresía en registro |
| **`resolveRegionFamily(cityName, countryId)`** | Consolidado; delega a `resolveEditorialFamily` |
| **`resolveRegionalPack(map, regionFamily, opts?)`** | Precedencia: explicit → `DEFAULT_FAMILY` → null (`resolvedFrom: missing`) |

Schema EFR: `3.8h.2-f2.2c-0.1`.

### 3.2 Eliminación fallbacks lógicos IBERIAN

**Antes (PRE-F2.2a):** 24 líneas ejecutables — `|| 'IBERIAN'`, `|| POOL.IBERIAN`, 3× `return 'IBERIAN'` local.

**Después (F2.2c):** **0** fallbacks lógicos en los 4 archivos editoriales. IBERIAN permanece como:

- `DEFAULT_FAMILY = 'IBERIAN'`
- Datos en tablas `IBERIAN: { ... }`
- Asignaciones explícitas en `COUNTRY_EDITORIAL_FAMILY` / `CITY_EDITORIAL_FAMILY`

### 3.3 Servicios migrados

| Servicio | Migración |
|----------|-----------|
| **Narrative** | `observeTailPool`, `humanizeTheme`, `buildNarrativeSummary` → `resolveRegionalPack` |
| **Composition** | 8 funciones pool + `editorialRegionalPack()` helper interno |
| **Knowledge** | `applyRegionalBlockVariation` → `resolveRegionalPack`; passthrough si pack ausente |

Las 3 copias locales de `resolveRegionFamily` **delegan a EFR** (sin `return 'IBERIAN'` hardcoded).

### 3.4 Fix IF-C08 — sparse metadata

**Problema:** `input.relevantInfluences || ctx.influences` pasaba `[]` truthy → `deriveNarrativeContext` fallaba con `no_influences` → `narrativeContext` null · `regionN` ausente.

**Fix:** usar `ctx.influences` cuando `input.relevantInfluences` está vacío; asignar `ctx.regionFamily` **una vez** vía EFR **antes** de composición narrative.

**Resultado verificado:** BA / trabajo / sparse → `regionN=LATAM` · `regionK=LATAM` · `ok:true` · `englishThemeHit:null`.
